# 【知らないと危険】Claude Code の危険なリスク5選とその対策

> チャンネル: Saix（杉田）
> 動画URL: https://youtu.be/aiek9fNCB00

---

## 概要

Claude Code は強力なCLI AIエージェントで、ファイルのフルアクセス・Git操作・Web取得・ブラウザ操作まで可能。
**何でもできるからこそリスク管理が極めて重要。**

実際に相談された事例として「データが消えて元に戻せない」「APIキーが漏洩した」「知らない間に高額請求された」などが多発している。

---

## リスク1：1,000万円の請求（青天井の課金）

### 何が危険か
- Claude Code は使った分だけ課金される**重量課金モデル**
- 上限設定なしで放置すると青天井で請求される
- AIに任せっきりにした結果、月末に**127万円の請求**が来た事例あり
- API連携ツールを開発した際、APIコストが裏側で積み上がるケースが多い（初心者に多い）

### 対策
| # | 対策 | 内容 |
|---|------|------|
| 1 | 支出の上限を設定 | コンソール → 設定 → 使用量 → 「上限を調整」から月間利用上限を設定。**「無制限」設定は絶対NG** |
| 2 | **Max Planに加入**（最推奨） | $100〜$200/月の定額プランに加入し、追加の重量課金をオフにする |
| 3 | `/cost` コマンドで常時監視 | セッション中のコスト・トークン使用量をリアルタイム確認する習慣をつける |

## リスク2：機密情報の全世界公開（情報漏洩）

### 何が危険か
- 「全部コミットして」と指示するだけで機密ファイルが GitHub に公開される
- `.gitignore` 未設定のまま公開リポジトリに push するリスク

### 危険な4ファイル（必ず管理）
| ファイル | 含まれる情報 |
|----------|-------------|
| `.env` | APIキー、データベース接続情報（漏洩 = クレジットカード盗難と同等） |
| `credentials.json` | 認証情報 |
| SSH秘密鍵 (`~/.ssh/id_rsa` 等) | 外部サーバーへの接続情報 |
| AWSアクセスキー (`~/.aws/credentials`) | AWSへの無制限アクセス権 |

### 対策
1. **`.gitignore` を必ず設定** — 機密ファイルを Git 管理から除外
2. **アクセス権限を制御** — 機密ファイルをクラウドに保存しようとしていないかチェックする仕組みを導入
3. **デフォルトモードを使用** — 編集・実行前に必ず人間が確認するモードで運用
4. **4層の防御** — 上記3つ＋定期的な棚卸しチェック

## リスク3：一瞬で全データ消去（破壊的コマンド）

### 何が危険か
- 「ディレクトリを整理して」「ファイルを綺麗にして」といった指示で、AIが破壊的コマンドを平気で実行する
- 実際に**元に戻せなくなった企業**から相談が来ている

### 絶対に使わせてはいけないコマンド
| コマンド | 被害 |
|----------|------|
| `rm -rf` | ファイル全削除（顧客リスト10年分・契約書PDFなど、PC内全データが消える） |
| `git push --force` | リモートを強制上書き・変更履歴の全消去 |
| `DROP TABLE` (DB) | データベースの全削除 |

### 対策
1. **Plan モード（読み取り専用）を使う** — 最も安全。実行前に計画のみ確認できる
2. **デフォルトモード** — 実行前に人間が必ず確認
3. **CLAUDE.md に禁止コマンドを明記** — 「rm -rf は絶対に使うな」と事前に指示
4. 「バイパスパーミッション・全て自動実行」系のコマンドは**絶対に使わない**

---

## リスク4：AIが乗っ取られる（プロンプトインジェクション）

### 何が危険か
- 攻撃者がファイルに**隠し指示を仕込み**、Claude Code がそのファイルを読み込むことで意図しない操作が行われる
- MCP サーバーや外部ツール連携を追加するほど攻撃面が拡大する

### 対策
1. **信頼できるソースのみ使用** — MCP・外部ツールは公式のものに限定
2. **デフォルトモード** — 実行前に人間が確認する仕組みを必ず維持
3. **外部通信を制限** — 不要なデータ送信をブロック
4. **ツール結果を目視確認** — 人間のチェックが最後の砦

---

## リスク5：脆弱性だらけのAI生成コード

### 何が危険か
- Claude Code は「動くコード」は書けるが、**「安全なコード」は別問題**
- テストなしでデプロイすると本番環境に脆弱性が公開され、ハッカーに悪用される

### 代表的な脆弱性
- SQL インジェクション
- 認証回避（Authentication Bypass）
- その他 OWASP Top 10 全般

### 対策
1. **コードレビューを必ず実施** — ユーザー入力・認証ロジックを重点的に確認
2. **人間によるレビュー** — AIが書いたコードをそのまま信用しない
3. **AIにセキュリティレビューを依頼** — 「OWASP Top 10の観点でレビューして」と具体的に指示
   - 「セキュリティチェックして」の一言だけでは**絶対にダメ**。脆弱性の種類まで明示すること

---

## まとめ：リスクと最重要対策一覧

| リスク | 最重要対策 |
|--------|-----------|
| 1,000万円請求 | Max Plan + 月間上限設定 + `/cost` 監視 |
| 機密情報漏洩 | `.gitignore` 設定 + `.env` 等の厳重管理 |
| データ全消去 | Plan モード / デフォルトモード + 禁止コマンドをCLAUDE.mdに明記 |
| プロンプトインジェクション | 信頼できるソースのみ + デフォルトモード |
| AI生成コードの脆弱性 | 具体的な脆弱性を指定したセキュリティレビュー依頼 |

> **ポイント：** 設定は一度やれば永続的に機能する。最初のセットアップを丁寧に行うことが最大のリスク回避になる。
