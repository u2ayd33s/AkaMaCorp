### 作業サマリー

MAREGroup Portal の全セクション（ヒーロー・掲示板・リンク集・社員紹介・アンケート）を REST API で動的生成する `SitePages__MAREGroup.js` を新規実装。mare-group サブサイトのリストからデータを取得し、テキストWebパーツのマーカー文言を検出して差し替え描画する。PR #4 でマージ完了。

---

### [決断] ニュースセクションを Search REST API からリスト REST API に変更
- 決断内容: `_api/search/query` ではなく `_api/web/lists/GetByTitle('サイトのページ')/items` + `$expand=MasterNewsCategory` を使用
- 理由: 検索スキーマにマネージドプロパティ（DisplayFlagOWSBOOL, MasterNewsCategoryOWSCHCS）が未登録で、Search API がカスタム列の値を返さなかった
- 結果: 参照列を $expand で直接展開でき、カテゴリタブ切替が正常動作

### [決断] CONFIG.LIST_SITE にサブサイト URL を固定
- 決断内容: リスト取得用に `https://u2ayd33s.sharepoint.com/sites/Share-Point/mare-group` を固定値として設定
- 理由: スクリプトは親サイト（Share-Point）の SitePages から読み込まれるが、リストは mare-group サブサイトに存在するため、getSiteUrl() では正しいパスを取得できなかった（404エラー）
- 結果: 全リスト取得が正常動作

### [決断] ニュース/ページの表示を CONFIG で ON/OFF 切替可能に
- 決断内容: `NEWS: true / PAGE: false` のフラグでPromotedState フィルタを動的構築する設計
- 理由: ユーザーからの提案。将来的にニュースとページを同時表示する要件に対応
- 結果: `(PromotedState eq 2 or PromotedState eq 1)` のような OData フィルタを動的生成

### [インサイト] SharePoint サイトのページの未作成カスタム列は $select に含めると 400 エラー
- 気づき: `DisplayFlag` 列がリストに存在しない状態で `$select=DisplayFlag` を指定すると 400 Bad Request になる。存在確認なしに $select へ列名を列挙するとエラーになる
- 適用場面: 新しいカスタム列を前提とした REST API クエリを書くとき

### [インサイト] Search API のマネージドプロパティはテナント管理者が検索スキーマに登録しないと取得不可
- 気づき: カスタム列は自動クロールされるが、マネージドプロパティとして `selectproperties` で指定するには検索スキーマへの明示的な登録が必要。特に参照列は Search API では扱いにくい
- 適用場面: Search API を使うか List API を使うかの判断時。参照列がある場合は List API + $expand が確実

### [インサイト] ScriptLoader スクリプトは SharePoint SPA ナビゲーションで二重実行される
- 気づき: SharePoint モダンページの SPA ナビゲーションにより `init()` が複数回呼ばれ、セクションが二重描画される。`_initialized` フラグ + `document.getElementById()` での既存コンテナチェックで防止する
- 適用場面: すべての ScriptLoader スクリプトの初期化処理

---

### [パターン] ScriptLoader 二重実行防止
```javascript
let _initialized = false;
function safeInit() {
  if (_initialized) return;
  _initialized = true;
  init();
}
// + replaceMarkerWithContainer 内で既存ID チェック
if (sectionId && document.getElementById(sectionId)) return null;
```
