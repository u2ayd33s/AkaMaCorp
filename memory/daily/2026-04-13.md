  ### 🤔 決断: AkaMaCorp フォルダをそのまま Obsidian Vault にする
  - 決断内容: 新リポジトリを作らず AkaMaCorp を Vault として開く
  - 理由: 既存の memory/ 構造がそのまま活きる。管理対象を増やさない
  - 結果: セットアップコストが最小で済んだ

  ### 🤔 決断: insights/errors を個別ファイルに分割する
  - 決断内容: insights.md / error-patterns.md を memory/insights/ memory/errors/ に移行
  - 理由: Dataview で status・date・タグによるフィルタリングが可能になる
  - 結果: dashboard.md で結晶化待ち/済み・未解決/解決済みを自動集計できた

  ### 💡 インサイト: CLAUDE.md をVaultルートに置くとClaude Codeが常時コンテキストを把握できる
  - 気づき: AkaMaCorp/CLAUDE.md を作成したことで、Claude Code がエージェント構成・memory運用ルールを毎回説明なしで把握できる
  - 適用場面: 今後のすべての作業セッション

  ### 💡 インサイト: デイリーノートへの記録を Claude Code の責務にする
  - 気づき: 作業終了時に Claude Code がデイリーノートを書くことで、記録が揮発せず Obsidian ナレッジベースに蓄積される。次回セッションのコンテキスト注入として機能する
  - 適用場面: personal-GitHub での全作業セッション終了時

  ### 💡 インサイト: wikilink追加でObsidianグラフビューが意味を持つ
  - 気づき: 全エージェントCLAUDE.md・agent-registry.md・root CLAUDE.mdにwikilink追加後、Obsidianグラフビューでエージェント→スキル→memoryの接続が視覚化された。frontmatterのtags/role/skillsによりDataviewクエリも可能になった
  - 適用場面: 新規ファイル作成時は必ずwikilink・frontmatterを付与する
