## DECISION
- SPFx カスタムHTML Webパーツの自動高さ調整方式: postMessageではなくsandboxなし＋contentDocument.scrollHeight直接計測を採用
  - 理由: SharePoint Online環境でsandbox iframeからのpostMessageがブロックされることが判明
- iframeのmin-height CSSを削除し、JS計測値のみで高さ制御する方式に変更
  - 理由: min-height CSSがJSの高さ設定を上書きし、小さいコンテンツでも100px余白が出てしまう問題を修正

## INSIGHT
- SPFx sandbox iframe + postMessageはSharePoint Online workbenchでブロックされる（CSPまたはフレーム制約）
- iframeの高さ縮小対応には「一旦1pxにリセット → scrollHeight計測 → 再設定」のパターンが有効
- srcdocで完全なHTML文書（DOCTYPE+head+body）を包まないとscrollHeightが正しく計算されない
- gulp package-solution --shipでEPERMエラーが出る場合はsolution/debugフォルダを手動削除してから再実行
- SPFx manifestのアイコンはofficeFabricIconFontNameをiconImageUrlに変えてbase64 data URLを設定できる

## PATTERN
- SPFx iframe自動高さ調整: sandbox外す → doc.write(fullDoc) → onload時にiframe.style.height='1px'→scrollHeight計測→再設定
- パッケージ化手順: gulp bundle --ship → rm -rf sharepoint/solution/debug → gulp package-solution --ship
