# エージェント ルーティング

ユーザーの指示を分析し、最適なエージェントへ割り振ります。

## 判断フロー

```
1. ユーザーの指示を読む
2. agent-registry.md を参照する
3. キーワード・文脈からエージェントを選定する
4. 複数必要な場合は順序を整理する
5. ユーザーに「◯◯ エージェントに依頼します」と伝えてから実行する
```

## ルーティング基準

### sharepoint エージェントを選ぶ場合
- SharePoint / SPFx / Microsoft 365 が絡む開発
- REST API / Graph API の実装
- SharePoint リスト・ライブラリの操作
- Web パーツ・拡張機能の作成

### frontend エージェントを選ぶ場合
- HTML / CSS / JavaScript の実装
- UI コンポーネント・ページデザインの作成
- レスポンシブ対応・モバイルデザイン
- UX 改善・アクセシビリティ対応

### 複数エージェントが必要な場合
SharePoint と Frontend の両方が必要な場合の実行順序:
```
① frontend → デザイン・UI を先に設計
② sharepoint → データ連携・バックエンドを実装
③ frontend → SharePoint データを組み込んだ最終実装
```

## 出力フォーマット

ルーティング結果は以下の形式でユーザーに伝える:

```
## タスク分析

**依頼内容**: [ユーザーの指示の要約]

**担当エージェント**: [エージェント名]
**使用スキル**: [スキル名]
**理由**: [なぜこのエージェントを選んだか]

**実行計画**:
1. [ステップ1]
2. [ステップ2]
...
```

## 判断できない場合

どのエージェントが適切か判断できない場合は、ユーザーに以下を確認する:
- 何を作りたいのか（成果物）
- どの技術・プラットフォームを使うか
- 優先順位（速さ vs 品質）
