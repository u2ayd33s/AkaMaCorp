# Skill: brand-guide

## 概要

プロジェクト・サービスのビジュアルアイデンティティを定義するブランドガイドラインを策定する。
カラー・タイポグラフィ・スペーシング・アイコン等のデザイントークンを体系化する。

## 使用場面

- 新規サービス・ページのデザイン基盤を作りたいとき
- 既存デザインのルールを文書化・統一したいとき
- デザイントークンを CSS 変数として定義したいとき
- チーム間でデザイン基準を共有したいとき

## ガイドライン構成

### カラーシステム
- プライマリカラー・セカンダリカラー
- グレースケールパレット
- セマンティックカラー（success / warning / error / info）
- テーマ対応（ライト / ダーク）

### タイポグラフィ
- フォントファミリー（日本語・英字）
- フォントサイズスケール（xs / sm / md / lg / xl / 2xl / ...）
- 行間・字間
- 見出し・本文・キャプションのスタイル定義

### スペーシングシステム
- ベース単位（4px / 8px グリッド）
- スペーシングスケール（4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / ...）

### コンポーネントスタイル
- ボーダー・角丸（border-radius）
- シャドウ（box-shadow）
- アイコンサイズ

## 手順

1. 既存デザイン・ブランド資料を収集
2. 各カテゴリのトークンを定義
3. Figma でスタイル / 変数（Variables）として登録
4. CSS 変数定義を出力（`:root { --color-primary: ...; }`）
5. 使用方法をドキュメントに記載

## 成果物

- ブランドガイドライン文書（Markdown）
- Figma スタイル / 変数定義
- CSS カスタムプロパティ定義ファイル

## SharePoint 環境での注意

SharePoint / Fluent UI v3 環境では既存の CSS 変数（`--colorBrandBackground` 等）と
競合しないよう、プロジェクト固有のプレフィックスを付与する（例: `--xsp-color-primary`）。
