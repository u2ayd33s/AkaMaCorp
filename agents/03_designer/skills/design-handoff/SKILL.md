# Skill: design-handoff

## 概要

Figmaデザインから開発（05_frontend / 04_sharepoint）向けの仕様書・実装指示を作成する。
デザイナーと開発者の認識齟齬を防ぎ、スムーズな実装を支援する。

## 使用場面

- Figmaデザインを開発エージェントに引き渡すとき
- デザインの詳細仕様（サイズ・色・余白・挙動）を文書化したいとき
- インタラクション・アニメーション仕様を定義するとき
- レスポンシブ対応の詳細を伝えるとき

## ハンドオフドキュメントの構成

### コンポーネント仕様
- コンポーネント名・用途
- プロパティ一覧（サイズ / 色 / テキスト / 状態）
- 使用する CSS 変数・デザイントークン
- 状態定義（default / hover / focus / active / disabled）

### レイアウト仕様
- グリッド・カラム数
- ブレークポイント（PC: 1024px+ / タブレット: 768px+ / モバイル: 375px+）
- コンテンツ幅・マージン・パディング

### インタラクション仕様
- クリック・ホバー時の変化
- トランジション・アニメーション（duration / easing）
- フォームバリデーション表示

### アセット
- アイコン・画像の書き出し仕様（SVG / PNG / WebP）
- ファイル命名規則

## 手順

1. Figma MCP (`figma-design` スキル) でデザインデータを取得
2. 各セクションの仕様をMarkdownで記述
3. CSS 変数・クラス名の対応表を作成
4. 開発エージェントへ引き渡し

## 成果物

- デザイン仕様書 (`DESIGN-SPEC.md`)
- コンポーネント仕様表
- アセット書き出しファイル
