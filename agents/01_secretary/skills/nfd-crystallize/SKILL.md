# 知識結晶化 (Knowledge Crystallization)

育成優先型開発 (Nurture-First Development) の核心メカニズム。
エクスペリエンス層に蓄積された断片的な知識をスキル層の構造化された知識アセットへ昇格させる。

## 結晶化の4フェーズ

```
① パターン抽出 (Pattern Extraction)
   → memory/ を走査し、[INSIGHT][ERROR][PATTERN] タグを収集する

② 知識構造化 (Knowledge Structuring)
   → 収集したパターンを Skill Layer 形式に整理する

③ 脱文脈化 (De-contextualization)
   → 状況固有の詳細を除去し、汎用的な原則に変換する

④ 統合 (Integration)
   → 検証済みのアセットを対象スキルの SKILL.md や references/ に書き込む
```

## 結晶化プロセス

### Step 1: スコープ決定

```markdown
対象期間: [前回 CC 以降]
対象エージェント: [全体 / 特定エージェント]
```

### Step 2: エクスペリエンス層の走査

以下のファイルを確認する:
- `memory/daily/*.md` — 日次ログ
- `memory/insights.md` — 洞察リスト
- `memory/error-patterns.md` — エラーパターン

### Step 3: パターン抽出と評価

各エントリを以下の基準で評価:
- **即時昇格可能**: `[INSIGHT]` で汎用性が高く検証済み
- **要蓄積**: `[ERROR]` で1件のみ（パターン確認が必要）
- **削除対象**: 陳腐化・誤りが判明したもの

### Step 4: スキル層への統合

| 結晶化先 | 内容 |
|---------|------|
| 対象スキルの `SKILL.md` | 新しい原則・ガイドライン・判断基準の追加 |
| `references/` サブファイル | 詳細な参照ドキュメント |
| `CLAUDE.md` | Constitutional Layer の更新（原則変更時のみ） |

### Step 5: 結晶化チェックポイント記録

`memory/crystallization/CC<N>-YYYY-MM-DD.md` を作成:
- 現在の知識状態スナップショット
- 生成した知識アセットのリスト
- 次回 CC までのアクション

## 結晶化トリガー

| トリガー種別 | 条件 |
|------------|------|
| **スケジュール** | 毎月16日 10:00 |
| **閾値** | 未結晶化ログ 20件超過 |
| **イベント** | 重要 PJ 完了、新エージェント追加、重大エラー発生 |

## 結晶化品質チェック

結晶化後、以下を確認する:
- [ ] 新しい原則は既存の Constitutional Layer と矛盾していないか
- [ ] 脱文脈化され、汎用的に適用できる記述になっているか
- [ ] エクスペリエンス層の元データで支持されているか（1件のみの事例で一般化していないか）
- [ ] ユーザーが確認できる形で報告したか

## 出力フォーマット

結晶化完了後のユーザーへの報告:

```

2. ...

### 見送り（要蓄積）
- [内容]: 理由

### 次回 CC の目安

```
