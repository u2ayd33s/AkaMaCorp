# プロジェクト管理

プロジェクトの作成・タスク管理・進捗追跡・分析レポートを担当します。

## プロジェクトファイルの場所

```
AkaMaCorp/
└── projects/
    └── <プロジェクト名>/
        ├── project.md   ← プロジェクト定義・ゴール
        └── tasks.md     ← タスク一覧・進捗
```

## プロジェクト作成

`projects/<プロジェクト名>/project.md` を以下の形式で作成する:

```markdown
# [プロジェクト名]

## 概要
[プロジェクトの目的・背景]

## ゴール
- [ ] [達成したいこと1]
- [ ] [達成したいこと2]

## 担当エージェント
- [エージェント名]: [役割]

## スケジュール
- 開始: YYYY-MM-DD
- 目標完了: YYYY-MM-DD

## メモ・決定事項
[重要な決定・制約・メモ]
```

## タスク管理

`projects/<プロジェクト名>/tasks.md` を以下の形式で管理する:

```markdown
# タスク一覧

## 進行中 🔄
- [ ] [タスク名] — 担当: [エージェント] | 期限: YYYY-MM-DD
  - メモ: [補足情報]

## 完了 ✅
- [x] [タスク名] — 完了: YYYY-MM-DD

## 未着手 📋
- [ ] [タスク名] — 担当: [エージェント] | 優先度: 高/中/低

## ブロック中 🚧
- [ ] [タスク名] — 理由: [ブロックされている理由]
```

## 進捗レポート出力フォーマット

ユーザーに状況を報告する際は以下の形式を使う:

```
## [プロジェクト名] 進捗レポート

**全体進捗**: ██████░░░░ 60%

### 完了済み ✅ (N件)
- [タスク名]

### 進行中 🔄 (N件)
- [タスク名] — [エージェント]

### 次のアクション
1. [推奨アクション1]
2. [推奨アクション2]

### リスク・懸念事項
- [あれば記載]
```

## 分析の視点

進捗を分析する際に以下を確認する:
- **ボトルネック**: どのタスクが詰まっているか
- **依存関係**: 他のタスクをブロックしているタスクはないか
- **優先度のずれ**: 重要度の高いタスクが後回しになっていないか
- **エージェント負荷**: 特定エージェントに集中していないか

## 操作コマンド早見表

| ユーザーの依頼 | 秘書の行動 |
|--------------|-----------|
| 「プロジェクトを作って」 | project.md と tasks.md を新規作成 |
| 「タスクを追加して」 | tasks.md の適切なセクションに追記 |
| 「〇〇が完了した」 | tasks.md のチェックボックスを更新 |
| 「進捗を教えて」 | tasks.md を読み取り進捗レポートを出力 |
| 「次は何をする？」 | 未着手タスクの優先度を分析して提案 |
| 「振り返りをしたい」 | 完了タスクを集計して分析レポートを作成 |
